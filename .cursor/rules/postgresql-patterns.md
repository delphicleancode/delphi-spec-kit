# PostgreSQL Database — Cursor Rules

Use estas regras ao desenvolver com banco de dados PostgreSQL via FireDAC.

## Conexão Obrigatória

```pascal
{ Configuração base }
FConnection.DriverName := 'PG';
FConnection.Params.Values['Server'] := 'localhost';
FConnection.Params.Values['Port'] := '5432';
FConnection.Params.Database := 'meubanco';
FConnection.Params.UserName := 'postgres';
FConnection.Params.Password := 'senha';
FConnection.Params.Values['CharacterSet'] := 'UTF8';
```

## Auto-Increment

| Abordagem | Versão | Exemplo |
|-----------|--------|---------|
| `SERIAL` | Todas | Legacy — cria sequence implícita |
| `BIGSERIAL` | Todas | Legacy — 64-bit |
| `GENERATED ALWAYS AS IDENTITY` | PG 10+ | ✅ Moderno — SQL Standard |
| `GENERATED BY DEFAULT AS IDENTITY` | PG 10+ | Permite override manual |
| `nextval('seq')` | Todas | Sequence manual |

## RETURNING — Mesma Regra do Firebird

```pascal
// ❌ ExecSQL perde o valor retornado!
LQuery.SQL.Text := 'INSERT INTO ... RETURNING id';
LQuery.ExecSQL;

// ✅ Open para receber RETURNING
LQuery.SQL.Text := 'INSERT INTO ... RETURNING id';
LQuery.Open;
LId := LQuery.FieldByName('id').AsInteger;
```

## UPSERT — INSERT ... ON CONFLICT

```pascal
LQuery.SQL.Text :=
  'INSERT INTO customers (cpf, name, email) ' +
  'VALUES (:cpf, :name, :email) ' +
  'ON CONFLICT (cpf) DO UPDATE SET ' +
  '  name = EXCLUDED.name, email = EXCLUDED.email ' +
  'RETURNING id';
LQuery.Open;
```

## JSONB — Dados Semi-Estruturados

```pascal
{ Gravar JSONB }
LQuery.ParamByName('settings').AsString := '{"theme":"dark"}';
{ Cast necessário no SQL: :param::jsonb }

{ Ler campo JSONB }
LQuery.SQL.Text := 'SELECT settings->>''theme'' FROM ... WHERE id = :id';
```

## Functions e Procedures

| Tipo | Chamada Delphi |
|------|----------------|
| **Function escalar** | `SELECT fn_nome(:param)` + `Open` |
| **Function RETURNS TABLE** | `SELECT * FROM fn_nome(:param)` + `Open` |
| **Procedure** (PG 11+) | `CALL sp_nome(:param)` + `ExecSQL` |

## Transactions

```pascal
{ Transação explícita }
FConnection.StartTransaction;
try
  FRepoA.Insert(LObjA);
  FRepoB.Insert(LObjB);
  FConnection.Commit;
except
  FConnection.Rollback;
  raise;
end;
```

| Isolation | FireDAC | Uso |
|-----------|---------|-----|
| Read Committed | `xiReadCommitted` | ✅ Padrão CRUD |
| Repeatable Read | `xiRepeatableRead` | Relatórios consistentes |
| Serializable | `xiSerializable` | Máxima consistência |

## Tratamento de Erros PostgreSQL

```pascal
except
  on E: EFDDBEngineException do
  begin
    case E.Kind of
      ekUKViolated:
        raise EDuplicateException.Create('Registro duplicado');
      ekFKViolated:
        raise EDependencyException.Create('Violação de FK');
      ekRecordLocked:
        raise EConflictException.Create('Registro bloqueado');
      ekServerGone:
        raise EConnectionLostException.Create('Conexão perdida');
    else
      raise;
    end;
  end;
end;
```

## Metadata — Verificar Existência

```sql
-- Tabela existe?
SELECT COUNT(*) FROM information_schema.tables
WHERE table_schema = 'public' AND table_name = 'customers';

-- Coluna existe?
SELECT COUNT(*) FROM information_schema.columns
WHERE table_schema = 'public' AND table_name = 'customers' AND column_name = 'email';

-- Sequence existe?
SELECT COUNT(*) FROM information_schema.sequences
WHERE sequence_schema = 'public' AND sequence_name = 'customers_id_seq';
```

## Proibições em PostgreSQL

- ❌ Concatenar SQL — usar parâmetros parametrizados
- ❌ `ExecSQL` com `RETURNING` — usar `Open`
- ❌ `SELECT *` em tabelas grandes — selecionar colunas necessárias
- ❌ N+1 queries (loop com query dentro) — usar JOIN/subquery
- ❌ `SERIAL` em novos projetos — usar `IDENTITY`
- ❌ Ignorar índices em colunas de WHERE/JOIN
- ❌ Ignorar `CharacterSet` na conexão
- ❌ Guardar JSON como TEXT — usar `JSONB`
- ❌ `RDB$RELATIONS` para metadata — usar `information_schema`
